# Lab 5.2: Search and Filter Implementation with State Management

## Обзор
Данная лабораторная работа расширяет концепции из Lab 5.1 и сосредоточена на реализации поиска и фильтрации данных. Реализован компонент `SearchApp`, который демонстрирует управление состоянием для функции поиска в реальном времени.

## Цель работы
Научиться управлять состоянием компонентов для реализации поиска и фильтрации данных, обрабатывать пользовательский ввод, обновлять состояние синхронно с действиями пользователя.

## Структура проекта в src:
*   `SearchApp.tsx` — Основной компонент с функцией поиска. Управляет состоянием поисковой строки и отфильтрованного списка пользователей. Демонстрирует использование `useState` для управления двумя связанными состояниями.
*   `UserCard.tsx` — Переиспользуемый компонент карточки пользователя из Lab 5.1.
*   `SkillList.tsx` — Компонент списка навыков из Lab 5.1.
*   `App.tsx` — Корневой компонент, демонстрирующий использование `SearchApp` и других компонентов.
*   `types.tsx` — Файл с TypeScript типами: `User` и `Skill`.

## Реализованный функционал
1.  **Real-Time Search:** Поиск срабатывает при каждом изменении значения в input поле благодаря обработчику `onChange`.
2.  **State Management:** Использование `useState` для управления `searchTerm` (значение в поле поиска) и `filteredUsers` (отфильтрованный список).
3.  **Filter Logic:** Фильтрация выполняется по имени пользователя с использованием `.filter()` и `.includes()` (case-insensitive).
4.  **Clear Function:** Кнопка "Очистить" позволяет сбросить поисковую строку и вернуть полный список пользователей.
5.  **Conditional Rendering:** Условное отображение списка результатов или сообщения об отсутствии результатов.
6.  **Component Composition:** Использование `UserCard` для отображения каждого результата поиска.

*   Исправлена структура JSX при выводе элементов в `.map()` (обёрнуты в div контейнер)
*   Исправлен регистр закрывающего тега `</UserCard>` на `</div>`

## Методология
Компонент демонстрирует паттерн управления состоянием для поиска данных. Ключевые концепции:
- Синхронизация input field с состоянием (`value={searchTerm}`)
- Обновление фильтрованного списка в зависимости от пользовательского ввода
- Использование методов массива для фильтрации данных
