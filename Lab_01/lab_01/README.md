# Lab 1.1: Declarative vs Imperative UI

## Обзор
В данной лабораторной работе проведено практическое сравнение двух парадигм программирования интерфейсов: императивной (на базе jQuery) и декларативной (на базе React). Основная задача — реализация функции переключения подсветки (highlight toggle).

## Цель работы
Продемонстрировать разницу в поддержке кода и его структуре. Понять, как декларативный подход React избавляет разработчика от прямого манипулирования DOM-деревом.

## Структура файлов:
*   `index.html` — Императивная реализация. Содержит HTML-разметку и скрипт на jQuery, который пошагово описывает процесс добавления/удаления CSS-класса.
*   `project_tasks/src/App.jsx` — Декларативная реализация внутри React-проекта. Описывает состояние интерфейса через `useState`.
*   `Task 3 : Frontend.pdf` — Аналитический отчет со сравнительным анализом масштабируемости обоих подходов.

## Методология
Работа выполнялась методом "параллельной реализации":
1.  **Императивный этап:** Использование селекторов и прямых команд (`.toggleClass()`) для изменения состояния элементов.
2.  **Декларативный этап:** Описание логики через состояние (state). Интерфейс обновляется автоматически ("реактивно") при изменении переменной `isHighlighted`.
3.  **Сравнение:** Оценка того, как декларативный подход масштабируется при усложнении интерфейса.
