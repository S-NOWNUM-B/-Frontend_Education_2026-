# Lab 7.1: React Router - Basic Routing and Navigation

## Обзор
Данная лабораторная работа посвящена изучению основ React Router. Реализовано многостраничное приложение с базовой навигацией, использующее Layout компонент с Outlet для отображения дочерних маршрутов.

## Цель работы
Научиться работать с React Router, создавать базовую структуру маршрутизации, использовать компоненты Link для навигации, применять Layout паттерн с Outlet для общей структуры страниц.

## Структура проекта в src:
*   `main.tsx` — Точка входа приложения. Настройка роутера с использованием `createBrowserRouter` и определение всех маршрутов.
*   `Layout.tsx` — Компонент обертки с навигацией и футером. Использует `Outlet` для рендеринга дочерних маршрутов.
*   `Home.tsx` — Главная страница приложения.
*   `Courses.tsx` — Страница со списком курсов.
*   `About.tsx` — Страница "О нас".
*   `NotFound.tsx` — Страница 404 для несуществующих маршрутов.

## Реализованный функционал
1.  **Basic Routing:** Настройка маршрутов с использованием `createBrowserRouter`.
2.  **Nested Routes:** Использование вложенных маршрутов через `children` в конфигурации роутера.
3.  **Layout Pattern:** Компонент `Layout` с навигацией и футером, использующий `Outlet` для отображения дочерних страниц.
4.  **Navigation Links:** Использование компонента `Link` для навигации между страницами без перезагрузки.
5.  **404 Handling:** Обработка несуществующих маршрутов с помощью wildcard маршрута (`*`).
6.  **Index Route:** Использование `index: true` для определения страницы по умолчанию на корневом маршруте.

## Ключевые концепции
*   **createBrowserRouter** - современный способ создания роутера в React Router v6+
*   **Outlet** - компонент-заполнитель для рендеринга дочерних маршрутов
*   **Link** - декларативная навигация без перезагрузки страницы
*   **Nested Routes** - иерархическая структура маршрутов для переиспользования Layout компонентов

## Методология
Приложение демонстрирует базовую архитектуру React Router с использованием Layout паттерна. Все страницы отображаются внутри общего Layout компонента, что обеспечивает консистентную структуру с навигацией и футером на всех страницах.

**Инструкция по запуску:**
1. Установите зависимости: `pnpm install`
2. Запустите проект: `pnpm dev`
